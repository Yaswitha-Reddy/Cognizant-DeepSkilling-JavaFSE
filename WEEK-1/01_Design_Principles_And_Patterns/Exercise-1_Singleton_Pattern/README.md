# Exercise 1: Implementing Singleton Pattern

## Aim
Implement the Singleton Design Pattern to ensure that only one instance of the Logger class exists throughout the application lifecycle.

## Pattern Type
Creational Design Pattern

## Description
The Singleton Pattern restricts object creation to a single instance and provides a global access point to that instance. In this exercise, a Logger class is implemented as a Singleton to maintain consistent logging across the application.

## Concepts Covered
- Singleton Pattern
- Private Constructor
- Static Variable
- Static Method
- Object Creation Control

- Logger.java
- SingletonTest.java
